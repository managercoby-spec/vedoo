## Step-by-Step Setup

### Phase 1: GitHub Setup (5 minutes)

1. **Create GitHub Repository**
   ```bash

   - Click "New Repository"
   - Name: veedco-platform
   - Make it Private
   - Don't add README (we have one)
   ```

2. **Push Your Code**
   ```bash
   cd veedco-platform
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/veedco-platform.git
   git push -u origin main
   ```

### Phase 2: Supabase Setup (10 minutes)

1. **Create Project**
   - Go to [supabase.com](https://supabase.com)
   - Click "New Project"
   - Choose organization
   - Name: `veedco-platform`
   - Database Password: (save this!)
   - Region: Choose closest to users
   - Click "Create"

2. **Run Database Schema**
   - Wait for project to finish setup
   - Go to SQL Editor (left sidebar)
   - Click "New Query"
   - Copy entire contents of `supabase-schema.sql`
   - Paste and click "Run"
   - Should see ‚úÖ "Success. No rows returned"

3. **Create Storage Buckets**
   
   Go to Storage (left sidebar) ‚Üí Create 4 buckets:
   
   **Bucket 1: videos**
   - Name: `videos`
   - Public: ‚úÖ Yes
   - File size limit: 100 MB
   - Allowed MIME types: `video/*`
   
   **Bucket 2: images**
   - Name: `images`
   - Public: ‚ùå No (user uploads)
   - File size limit: 10 MB
   - Allowed MIME types: `image/*`
   
   **Bucket 3: thumbnails**
   - Name: `thumbnails`
   - Public: ‚úÖ Yes
   - File size limit: 5 MB
   - Allowed MIME types: `image/*`
   
   **Bucket 4: avatars**
   - Name: `avatars`
   - Public: ‚úÖ Yes
   - File size limit: 2 MB
   - Allowed MIME types: `image/*`

4. **Configure Auth Providers**
   
   Go to Authentication ‚Üí Providers:
   
   **Email (Already enabled by default)**
   - Enable email confirmations if wanted
   
   **Google OAuth (Optional but recommended)**
   - Enable Google
   - Go to [Google Cloud Console](https://console.cloud.google.com)
   - Create new project
   - Enable Google+ API
   - Create OAuth 2.0 credentials
   - Add authorized redirect URI: 
     `https://YOUR_PROJECT.supabase.co/auth/v1/callback`
   - Copy Client ID and Secret to Supabase

5. **Get API Keys**
   - Go to Project Settings (gear icon) ‚Üí API
   - Copy these values:
     - `Project URL` ‚Üí NEXT_PUBLIC_SUPABASE_URL
     - `anon public` key ‚Üí NEXT_PUBLIC_SUPABASE_ANON_KEY
     - `service_role` key ‚Üí SUPABASE_SERVICE_ROLE_KEY
   - ‚ö†Ô∏è **NEVER commit service_role key to GitHub!**

### Phase 3: Vercel Setup (5 minutes)

1. **Import GitHub Repository**
   - Go to [vercel.com](https://vercel.com)
   - Click "Add New..." ‚Üí "Project"
   - Import your `veedco-platform` repo
   - Click "Import"

2. **Configure Project**
   - Framework Preset: **Next.js** (auto-detected)
   - Root Directory: `./`
   - Build Command: `npm run build` (default)
   - Output Directory: `.next` (default)

3. **Add Environment Variables**
   
   Click "Environment Variables" and add:
   
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGc...
   SUPABASE_SERVICE_ROLE_KEY=eyJhbGc...
   NEXT_PUBLIC_SITE_URL=https://your-project.vercel.app
   ```
   
   ‚ö†Ô∏è Make sure to select **All** (Production, Preview, Development)

4. **Deploy**
   - Click "Deploy"
   - Wait 2-3 minutes
   - Your site is live! üéâ

### Common Issues & Solutions

#### ‚ùå "Invalid API key"
- Check `.env.local` has correct keys
- Restart dev server after changing env vars
- Keys should not have quotes around them

#### ‚ùå "Row Level Security" errors
- Make sure you ran the complete SQL schema
- Check RLS policies are enabled
- User must be authenticated

#### ‚ùå Build fails on Vercel
- Check environment variables are set correctly
- Make sure all dependencies are in package.json
- Check build logs for specific error

#### ‚ùå "Storage bucket not found"
- Create all 4 buckets in Supabase Storage
- Names must match exactly: videos, images, thumbnails, avatars

#### ‚ùå Can't upload images
- Check bucket exists
- Check bucket is public/private as needed
- Check file size within limits
- Check MIME types allowed
